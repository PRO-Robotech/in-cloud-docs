2.3. Network

2.3.1. Network Infrastructure Management System Requirements​

No.Requirement
1The platform must support network resource virtualization technologies (SDN)
2The platform must support centralized network resource management (SDN)
3The platform must support VLAN technologies
4The platform must support Overlay Network technologies
5The platform must support SNAT technologies
6The platform must support DNAT technologies
7The platform must support traffic encryption technologies
8The platform must support DHCP server and DHCP relay technology
9The platform must support traffic management prioritization technologies (QoS)
10The platform must support traffic balancing technologies at L4/L7 levels of the OSI model
11The platform must support virtual firewall technologies implementing ACL at OSI model layer 3 — Labels functionality at subnet level
12The platform must support virtual firewall technologies implementing ACL at OSI model layer 4 — Labels functionality at subnet level
13The platform must support command-line interface (CLI) management of SDN components, including network load balancers.
14The platform must support graphical user interface (GUI) management of SDN components, including network load balancers.
15The platform must have an API for management and integration with SDN components, including network load balancers.
16The platform must support granular traffic export and its characteristics (Netflow, SPAN) with detail down to the interface level (virtual interface)
17The platform must support archiving and restoring SDN settings
18The platform must support fault-tolerant deployment of SDN components (Controllers, virtual switches, etc.)
19The platform must support gRPC traffic on the Ingress controller

2.3.2. Network Infrastructure Requirements​

No.Requirement
1The platform must support dynamic routing protocols
2The platform must support Jumbo Frames of at least 9100 bytes
3The platform must support VM connection to SDN at 10 Gbps between two or more data centers
4The platform must support VM connection to SDN at 25 Gbps within a single data center
5The platform must support internal SDN network switching/routing performance of 10 Gbps and above
6The platform must support external network connection performance in L2out/L3out mode of 10 Gbps and above
7The platform must support subnet and IP address lifecycle management (tracking, allocation, release)
8The platform must provide security policy management for network connections
9The platform must support automatic network policy configuration for new subnets
10The platform must support VPN tunnel management (creation, configuration, deletion)
11The platform must support NAT rule management (creation, configuration, deletion)
12The platform must support virtual router and load balancer configuration management
13The platform must provide integration with network infrastructure monitoring systems
14The platform must support virtual firewall management (creation, configuration, modification, deletion)
15The platform must support network resource reservation management (IP, subnets, routes)
