2.8. Monitoring

2.8.1. Basic Monitoring Requirements​

No.Requirement
1The platform must provide the ability to monitor availability, collect, and display data on the state of all management components/services.
2The platform must provide the ability for proactive and event-driven monitoring of the cloud platform infrastructure, identifying bottlenecks and potential issues.
3The platform must provide the ability to collect data and metrics on cloud platform component utilization.
4The platform must support metric collection broken down by platform components, VMs, and services deployed on the platform.
5The platform must support CPU utilization metric collection for compute resource components.
6The platform must support RAM utilization metric collection for compute resource components.
7The platform must support network interface performance and utilization metric collection for compute resource components.
8The platform must support disk subsystem utilization metric collection by volume for compute resource components.
9The platform must support CPU utilization metric collection for VMs (services) deployed on the platform.
10The platform must support RAM utilization metric collection by volume for VMs (services) deployed on the platform.
11The platform must support network interface performance and utilization metric collection for VMs (services) deployed on the platform.
12The platform must support disk subsystem utilization metric collection for VMs (services) deployed on the platform.
13The platform must support the ability to display collected data and metrics on self-service and administration portals.
14The platform must support the ability to manage and display monitoring parameters through a graphical user interface (GUI).
15Integration with banking monitoring systems is required.
16Providing alerting based on specified parameters.
17The platform must support the ability to configure event, trigger, and status notifications via email.
18The platform must allow configuring protection against data overflow, including data cleanup.
