2.7. Portal

2.7.1. General Interface Requirements​

No.Requirement
1The portal must be implemented through a graphical web interface.int +out +
2The platform must support service provisioning in multiple virtualization clusters from a single console with a graphical interface.
2The portal must be accessible in various browsers (Google Chrome, Edge, Firefox).
3The portal must have a responsive interface for different screen sizes to work on various devices.
4The portal must support a multilingual interface.
5The portal must implement user notification functionality within the platform interface.
6The portal must support multitenancy functionality and provide RBAC-based object visibility management.
7The portal must provide an access interface for managing allocated resources, including virtual machine consoles.
8The portal must have the ability to customize styling according to the customer's corporate template.

2.7.2.2. General Service Catalog Requirements​

No.Requirement
1The service catalog must support creating services based on pre-built VM images.
2The service catalog must support creating services for software deployed in k8s containers.
3The service catalog must support creating services based on k8s container images or VMs with the ability to subsequently add and configure necessary software using automation technologies (Ansible, Terraform).
4The service catalog must support providing SaaS applications.
5The service catalog must support service publication/archiving.
6The service catalog must support software version management when ordering a service.
7The platform must support ordering services from the service catalog.
8The platform must support integration with workflow systems for implementing the service approval process within the platform.
9The platform must support integration with workflow systems for implementing the service deployment process within the platform.
10The service catalog must support service search and filtering by attributes/tags.
11The service catalog must support service categorization.

2.7.3. Administration Section Requirements​

No.Requirement
1The administration portal must provide an interface for displaying information about cloud infrastructure objects.
2The administration portal must provide an interface for displaying reports on cloud infrastructure object usage: current and historical utilization.
3The administration portal must provide an interface for managing roles and access to the platform/projects (including RBAC-based).
4The administration portal must provide an interface for managing the lifecycle of internal cloud user accounts.
5The administration portal must provide an interface for managing projects (creation, modification, archiving).
6The administration portal must provide an interface for managing resource quotas.
7The administration portal must provide an interface for managing billing for projects.
8The administration portal must provide an interface for displaying project information (resources, quotas, tariffs, balance, etc.).
9The administration portal must provide an interface for displaying catalog service statuses.
10The administration portal must provide an interface for managing the service catalog.
11The administration portal must support template and image management for OS, IaaS, PaaS, SaaS services, etc.

2.7.4. Interface and Training Requirements​

No.Requirement
1Supports user interface localization (Russian, Kazakh, English).
2Availability of a demonstration stand for training.
3Training courses/videos on working with the product are available.
4Expected system learning time for a user does not exceed 10 hours.
5A user interface for administration is available.
6Input data validation functionality is available.
7The user interface can be customized according to the bank's design system.
8A built-in help/hint system is available to assist beginner users.
9Interactive interface elements must be visually distinguished from inactive ones.
10The system allows configuring tooltips that appear when hovering over an interface element.
11The system must correctly handle simultaneous modification conflicts of the same object by multiple users.
12All user-made settings must be saved individually for each authenticated user.
13Administrative user and role settings must be centralized (collected in a single system interface).
14Functionally similar interactive interface elements must look uniform throughout the System.
