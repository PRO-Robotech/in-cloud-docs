5.1.5.7. Restoration

In this chapter, we will consider options for restoring ETCD.

Note

This section depends on the following sections:

  • Backups.
  • etcdbrctl
Instructions
Note

The restore script must be run on all three nodes. Environment variables also need to be prepared on each node before running.

Environment variables

# General
export NODE_NAME_1="master-1"; export NODE_IP_1="10.0.0.11"
export NODE_NAME_2="master-2"; export NODE_IP_2="10.0.0.12"
export NODE_NAME_3="master-3"; export NODE_IP_3="10.0.0.13"

export S3_REGION=ru1
export S3_ACCESS_KEY_ID=""
export S3_SECRET_ACCESS_KEY=""
export S3_ENDPOINT=https://example.com
export S3_FORCE_PATH_STYLE=true
export S3_BUCKET=etcd-backups
export S3_PREFIX=cluster-name
  1. Run the restore script created in section 5.1.1.2
bash /etc/kubernetes/etcd/backup/restore.sh
