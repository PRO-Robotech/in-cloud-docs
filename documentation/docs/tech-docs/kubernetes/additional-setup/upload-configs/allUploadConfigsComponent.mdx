> Этот раздел посвящён загрузке актуальной конфигурации `kubeadm` и `kubelet` в кластер в виде `ConfigMap`. Такая конфигурация необходима для корректного выполнения команды `kubeadm join`, так как она используется при инициализации новых узлов управляющего контура.
> Загрузка конфигурации централизует управление параметрами кластера и обеспечивает единообразие между всеми нодами, включая как `master`, так и `worker`-узлы.

      При добавлении нового узла управляющего контура (`join`) загрузка конфигурации выполняется автоматически
      в рамках фазы `kubeadm join`. Конфигурация `kubeadm` и `kubelet` считывается из уже существующих `ConfigMap`
      в namespace `kube-system`, которые были загружены на этапе инициализации первого узла.

      Отдельного ручного вызова `upload-config` при join не требуется — `kubeadm join` самостоятельно
      получает необходимые параметры из кластера.
