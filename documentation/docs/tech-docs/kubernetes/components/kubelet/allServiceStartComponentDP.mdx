> На этом шаге нода получает все, что нужно для старта `kubelet`, и регистрируется в кластере.
> В ручном сценарии собираем конфиг и поднимаем сервис сами, а в варианте с `kubeadm` достаточно одной команды.

      :::warning Предварительные требования
      Перед стартом `kubelet` убедитесь, что вы уже прошли раздел
      [5.3.2. Аутентификация](/tech-docs/kubernetes/certificates/authentication-dp) и получили:
      - Получение CA-сертификата (`ca.crt`)
      - Создание `bootstrap-kubelet.conf` (или сертификатов вручную)
      :::

      <KubeletSystemdUnitStart />

    <TabItem value="Kubeadm">

      > Здесь достаточно выполнить `kubeadm join`: команда сама поднимет `kubelet`,
      > выполнит TLS Bootstrap и зарегистрирует ноду.

      :::warning Обратите внимание
      Перед запуском команды должен быть готов конфиг из раздела
      [Настройка ПО (DP)](/tech-docs/kubernetes/components/all-setup-dp).
      :::

      <CodeBlock language="bash">
        {dedent`
          kubeadm join \\
            --config=${CUSTOM_VALUE.kubeadmBaseConfigPath.value}/kubeadm.yaml
        `}
      </CodeBlock>

      :::note Вывод команды
