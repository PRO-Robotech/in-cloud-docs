# 5.2.1.3.5.1. Init Manifests

:::warning
Обратите внимание, что данный раздел описывает процесс генерации манифестов для инициализации кластера Kubernetes.
:::

## 5.2.1.3.5.1.1. Инициализация УК

<h4>Переменные окружния</h4>
<CodeBlock language="bash">
    {dedent`
        export BASE_K8S_PATH="${CUSTOM_VALUE.kuberneteBaseFolderPath.value}"
    `}
</CodeBlock>

<h4>Генерация манифеста</h4>
<CodeBlock language="bash">
    {dedent`
        kubeadm init phase control-plane all --config=${CUSTOM_VALUE.kuberneteBaseFolderPath.value}/kubeadm.conf
    `}
</CodeBlock>


:::note
<CodeBlock language="bash">
    {dedent`
        [control-plane] Using manifest folder "/etc/kubernetes/manifests"
        [control-plane] Creating static Pod manifest for "kube-apiserver"
        [control-plane] Creating static Pod manifest for "kube-controller-manager"
        [control-plane] Creating static Pod manifest for "kube-scheduler"
    `}
</CodeBlock>
:::

## 5.2.1.3.5.1.2. Инициализация ETCD

<h4>Генерация манифеста</h4>
<CodeBlock language="bash">
    {dedent`
        kubeadm init phase etcd local --config=${CUSTOM_VALUE.kuberneteBaseFolderPath.value}/kubeadm.conf
    `}
</CodeBlock>


:::note
<CodeBlock language="bash">
    {dedent`
        [etcd] Creating static Pod manifest for local etcd in "/etc/kubernetes/manifests"
    `}
</CodeBlock>
:::
