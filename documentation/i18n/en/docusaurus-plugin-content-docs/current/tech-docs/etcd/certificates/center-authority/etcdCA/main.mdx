> **Purpose:** CA for all etcd cluster certificates. Signs server, client, and peer certificates for etcd: `etcd-server` (client connections, port 2379), `etcd-peer` (inter-node replication, port 2380), and `etcd-healthcheck-client` (health checks). Also used by kube-apiserver to verify the connection to etcd via the `apiserver-etcd-client` certificate.

:::warning Note
**Note:** this section only describes the process of creating ```ETCD CA``` root certificates.
:::
