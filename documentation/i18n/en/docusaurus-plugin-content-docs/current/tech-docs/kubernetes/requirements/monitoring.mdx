# 2.8. Monitoring
## 2.8.1. Basic Monitoring Requirements


<table>
  <thead>
    <tr>
      <th>No.</th>
      <th>Requirement</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1</td>
      <td>The platform must provide the ability to monitor availability, collect, and display data on the state of all management components/services.</td>
    </tr>
    <tr>
      <td>2</td>
      <td>The platform must provide the ability for proactive and event-driven monitoring of the cloud platform infrastructure, identifying bottlenecks and potential issues.</td>
    </tr>
    <tr>
      <td>3</td>
      <td>The platform must provide the ability to collect data and metrics on cloud platform component utilization.</td>
    </tr>
    <tr>
      <td>4</td>
      <td>The platform must support metric collection broken down by platform components, VMs, and services deployed on the platform.</td>
    </tr>
    <tr>
      <td>5</td>
      <td>The platform must support CPU utilization metric collection for compute resource components.</td>
    </tr>
    <tr>
      <td>6</td>
      <td>The platform must support RAM utilization metric collection for compute resource components.</td>
    </tr>
    <tr>
      <td>7</td>
      <td>The platform must support network interface performance and utilization metric collection for compute resource components.</td>
    </tr>
    <tr>
      <td>8</td>
      <td>The platform must support disk subsystem utilization metric collection by volume for compute resource components.</td>
    </tr>
    <tr>
      <td>9</td>
      <td>The platform must support CPU utilization metric collection for VMs (services) deployed on the platform.</td>
    </tr>
    <tr>
      <td>10</td>
      <td>The platform must support RAM utilization metric collection by volume for VMs (services) deployed on the platform.</td>
    </tr>
    <tr>
      <td>11</td>
      <td>The platform must support network interface performance and utilization metric collection for VMs (services) deployed on the platform.</td>
    </tr>
    <tr>
      <td>12</td>
      <td>The platform must support disk subsystem utilization metric collection for VMs (services) deployed on the platform.</td>
    </tr>
    <tr>
      <td>13</td>
      <td>The platform must support the ability to display collected data and metrics on self-service and administration portals.</td>
    </tr>
    <tr>
      <td>14</td>
      <td>The platform must support the ability to manage and display monitoring parameters through a graphical user interface (GUI).</td>
    </tr>
    <tr>
      <td>15</td>
      <td>Integration with banking monitoring systems is required.</td>
    </tr>
    <tr>
      <td>16</td>
      <td>Providing alerting based on specified parameters.</td>
    </tr>
    <tr>
      <td>17</td>
      <td>The platform must support the ability to configure event, trigger, and status notifications via email.</td>
    </tr>
    <tr>
      <td>18</td>
      <td>The platform must allow configuring protection against data overflow, including data cleanup.</td>
    </tr>
  </tbody>
</table>
