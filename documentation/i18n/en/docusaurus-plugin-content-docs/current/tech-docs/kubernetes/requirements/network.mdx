# 2.3. Network
## 2.3.1. Network Infrastructure Management System Requirements

<table>
  <thead>
    <tr>
      <th>No.</th>
      <th>Requirement</th>
    </tr> 
  </thead>
  <tbody>
    <tr>
      <td>1</td>
      <td>The platform must support network resource virtualization technologies (SDN)</td>
    </tr>
    <tr>
      <td>2</td>
      <td>The platform must support centralized network resource management (SDN)</td>
    </tr>
    <tr>
      <td>3</td>
      <td>The platform must support VLAN technologies</td>
    </tr>
    <tr>
      <td>4</td>
      <td>The platform must support Overlay Network technologies</td>
    </tr>
    <tr>
      <td>5</td>
      <td>The platform must support SNAT technologies</td>
    </tr>
    <tr>
      <td>6</td>
      <td>The platform must support DNAT technologies</td>
    </tr>
    <tr>
      <td>7</td>
      <td>The platform must support traffic encryption technologies</td>
    </tr>
    <tr>
      <td>8</td>
      <td>The platform must support DHCP server and DHCP relay technology</td>
    </tr>
    <tr>
      <td>9</td>
      <td>The platform must support traffic management prioritization technologies (QoS)</td>
    </tr>
    <tr>
      <td>10</td>
      <td>The platform must support traffic balancing technologies at L4/L7 levels of the OSI model</td>
    </tr>
    <tr>
      <td>11</td>
      <td>The platform must support virtual firewall technologies implementing ACL at OSI model layer 3 — Labels functionality at subnet level</td>
    </tr>
    <tr>
      <td>12</td>
      <td>The platform must support virtual firewall technologies implementing ACL at OSI model layer 4 — Labels functionality at subnet level</td>
    </tr>
    <tr>
      <td>13</td>
      <td>The platform must support command-line interface (CLI) management of SDN components, including network load balancers.</td>
    </tr>
    <tr>
      <td>14</td>
      <td>The platform must support graphical user interface (GUI) management of SDN components, including network load balancers.</td>
    </tr>
    <tr>
      <td>15</td>
      <td>The platform must have an API for management and integration with SDN components, including network load balancers.</td>
    </tr>
    <tr>
      <td>16</td>
      <td>The platform must support granular traffic export and its characteristics (Netflow, SPAN) with detail down to the interface level (virtual interface)</td>
    </tr>
    <tr>
      <td>17</td>
      <td>The platform must support archiving and restoring SDN settings</td>
    </tr>
    <tr>
      <td>18</td>
      <td>The platform must support fault-tolerant deployment of SDN components (Controllers, virtual switches, etc.)</td>
    </tr>
    <tr>
      <td>19</td>
      <td>The platform must support gRPC traffic on the Ingress controller</td>
    </tr>
  </tbody>
</table>


## 2.3.2. Network Infrastructure Requirements

<table>
  <thead>
    <tr>
      <th>No.</th>
      <th>Requirement</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1</td>
      <td>The platform must support dynamic routing protocols</td>
    </tr>
    <tr>
      <td>2</td>
      <td>The platform must support Jumbo Frames of at least 9100 bytes</td>
    </tr>
    <tr>
      <td>3</td>
      <td>The platform must support VM connection to SDN at 10 Gbps between two or more data centers</td>
    </tr>
    <tr>
      <td>4</td>
      <td>The platform must support VM connection to SDN at 25 Gbps within a single data center</td>
    </tr>
    <tr>
      <td>5</td>
      <td>The platform must support internal SDN network switching/routing performance of 10 Gbps and above</td>
    </tr>
    <tr>
      <td>6</td>
      <td>The platform must support external network connection performance in L2out/L3out mode of 10 Gbps and above</td>
    </tr>
    <tr>
      <td>7</td>
      <td>The platform must support subnet and IP address lifecycle management (tracking, allocation, release)</td>
    </tr>
    <tr>
      <td>8</td>
      <td>The platform must provide security policy management for network connections</td>
    </tr>
    <tr>
      <td>9</td>
      <td>The platform must support automatic network policy configuration for new subnets</td>
    </tr>
    <tr>
      <td>10</td>
      <td>The platform must support VPN tunnel management (creation, configuration, deletion)</td>
    </tr>
    <tr>
      <td>11</td>
      <td>The platform must support NAT rule management (creation, configuration, deletion)</td>
    </tr>
    <tr>
      <td>12</td>
      <td>The platform must support virtual router and load balancer configuration management</td>
    </tr>
    <tr>
      <td>13</td>
      <td>The platform must provide integration with network infrastructure monitoring systems</td>
    </tr>
    <tr>
      <td>14</td>
      <td>The platform must support virtual firewall management (creation, configuration, modification, deletion)</td>
    </tr>
    <tr>
      <td>15</td>
      <td>The platform must support network resource reservation management (IP, subnets, routes)</td>
    </tr>
  </tbody>
</table>
