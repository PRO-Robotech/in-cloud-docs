# 2.9. Logging
## 2.9.1. Basic Logging Requirements

<table>
  <thead>
    <tr>
      <th>No.</th>
      <th>Requirement</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1</td>
      <td>The platform must provide event logging for all cloud infrastructure components (date and time of event, identifier, object, event result), including configuration changes of cloud platform components, as well as VMs and services deployed on the platform</td>
    </tr>
    <tr>
      <td>2</td>
      <td>The platform must support event severity level classification mechanisms (Critical, Warning, Informational)</td>
    </tr>
    <tr>
      <td>3</td>
      <td>The platform must support log protection mechanisms against modifications</td>
    </tr>
    <tr>
      <td>4</td>
      <td>The platform must support logging level configuration mechanisms for each cloud platform component</td>
    </tr>
    <tr>
      <td>5</td>
      <td>The platform must support cross-component event search across all cloud platform components</td>
    </tr>
    <tr>
      <td>6</td>
      <td>The platform must support log display filtering by event types, objects, severity levels</td>
    </tr>
    <tr>
      <td>7</td>
      <td>The platform must support collection and centralized storage of data, events, and metrics about the state of software and hardware components of the cloud platform</td>
    </tr>
  </tbody>
</table>
