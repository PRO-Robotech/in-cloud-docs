# 2.2. Containerization

## 2.2.1. Container Management System Requirements

<table>
  <thead>

      <th>No.</th>

  </thead>

      <td>Ability to deploy HA Kubernetes clusters on physical servers (bare-metal) in 2 or more data centers</td>

    <tr>

      <td>Ability to manage multiple Kubernetes clusters deployed at different geographic locations from a single interface</td>

    </tr>
    <tr>
      <td>5</td>

    </tr>
    <tr>
      <td>6</td>
      <td>Support for external load balancing in Kubernetes clusters</td>
    </tr>
    <tr>
      <td>7</td>
      <td>Support for internal load balancing in Kubernetes clusters</td>
    </tr>
    <tr>
      <td>8</td>
      <td>Ability to use load balancers in high availability mode</td>
    </tr>
    <tr>
      <td>9</td>
      <td>Ability to use a reverse proxy with load balancing function and orchestration API integration capability</td>
    </tr>
    <tr>
      <td>10</td>
      <td>Ability to integrate Ingress controller with load balancers</td>
    </tr>
    <tr>
      <td>11</td>
      <td>Kubernetes cluster node scanning function</td>
    </tr>
    <tr>
      <td>12</td>
      <td>Running container scanning function</td>
    </tr>
    <tr>
      <td>13</td>
      <td>External container image registry scanning function</td>
    </tr>
    <tr>
      <td>14</td>
      <td>Certificate management function availability</td>
    </tr>
    <tr>
      <td>15</td>
      <td>Ability to create custom security policies at the running application level</td>
    </tr>
    <tr>
      <td>16</td>
      <td>Container security management through a single, centralized web interface</td>
    </tr>
    <tr>
      <td>17</td>
      <td>Ability to export data/logs/reports on containerization platform security status for a specific time period</td>
    </tr>
    <tr>
      <td>18</td>
      <td>Availability of a recommendation system for vulnerability/incorrect/unwanted configuration remediation</td>
    </tr>
    <tr>
      <td>19</td>
      <td>Availability of ready-made CNI plugins for container network interface management</td>
    </tr>
    <tr>
      <td>20</td>
      <td>Ability to connect Persistent Volume</td>
    </tr>
    <tr>
      <td>21</td>
      <td>Ability to automate deployment and scaling of the containerization platform from the management interface without manually creating objects and editing configurations (for Platform administrator)</td>
    </tr>
    <tr>
      <td>22</td>
      <td>Ability to automate deployment and scaling of a project or containerization project instance from the management interface without manually creating objects and editing configurations (for Platform consumer)</td>
    </tr>
    <tr>
      <td>23</td>
      <td>Availability of a built-in CSI driver for the containerization platform in use</td>
    </tr>

    <tr>

    <tr>
