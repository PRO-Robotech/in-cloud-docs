> This section describes connecting a worker node to a Kubernetes cluster and starting Kubelet.
> For manual installation (Hard Way), you need to create a bootstrap kubeconfig with an authentication token,
> a base kubelet configuration file, and start the systemd service.
> When using kubeadm, simply run the `kubeadm join` command.

      <h4>Bootstrap kubeconfig</h4>

      > To authenticate the kubelet on a worker node, a bootstrap kubeconfig with a token is required.
      > After starting, the kubelet will automatically perform TLS Bootstrap and obtain a client certificate.

      <CodeBlock language="bash">
        {dedent`
          export BOOTSTRAP_TOKEN=${CUSTOM_VALUE.bootstrapToken.value}
        `}
      </CodeBlock>

      <CodeBlock language="bash">
        {dedent`
          cat <<EOF > ${CUSTOM_VALUE.kubernetesBaseFolderPath.value}/bootstrap-kubelet.conf
          apiVersion: v1
          clusters:
          - cluster:
              certificate-authority-data: $(base64 -w 0 ${CUSTOM_VALUE.kubernetesBaseFolderPath.value}/pki/ca.crt)
              server: https://api.${CUSTOM_VALUE.clusterName.value}.${CUSTOM_VALUE.kubernetesBaseDomain.value}:6443
            name: ${CUSTOM_VALUE.clusterName.value}
          contexts:
          - context:
              cluster: ${CUSTOM_VALUE.clusterName.value}
              user: tls-bootstrap-token-user
            name: tls-bootstrap-token-user@kubernetes
          current-context: tls-bootstrap-token-user@kubernetes
          kind: Config
          preferences: {}
          users:
          - name: tls-bootstrap-token-user
            user:
              token: $\{BOOTSTRAP_TOKEN}
          EOF
        `}
      </CodeBlock>
