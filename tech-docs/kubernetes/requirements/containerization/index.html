2.2. Контейнеризация

2.2.1. Требования к системе управления контейнерами​

№Требование
1Возможность развертывания HA-кластеров Kubernetes на виртуальных машинах в 2-х и более ЦОДах
2Возможность развертывания HA-кластеров Kubernetes на физических серверах (bare-metal) в 2-х и более ЦОДах
3Возможность управления множеством кластеров Kubernetes, развернутыми на разных географических площадках, из единого интерфейса
4Поддержка масштабирования узлов кластера Kubernetes в автоматическом режиме
5Поддержка вертикального и горизонтального масштабирования подов в автоматическом режиме
6Поддержка внешней балансировки нагрузки в кластерах Kubernetes
7Поддержка внутренней балансировки нагрузки в кластерах Kubernetes
8Возможность использования балансировщиков в режиме высокой доступности
9Возможность использования обратного прокси с функцией балансировки нагрузки и возможностью интеграции с API оркестрации
10Возможность интеграции Ingress контроллера с балансировщиками нагрузки
11Функция сканирования нод кластеров Kubernetes
12Функция сканирования запущенных контейнеров
13Функция сканирования внешних, подключаемых хранилищ образов контейнеров
14Наличие функции управления сертификатами
15Наличие возможности создания собственных политик безопасности на уровне запускаемых приложений
16Управление безопасностью контейнеров через единый, централизованный веб-интерфейс
17Наличие возможности выгрузки данных/логов/отчётов о состоянии безопасности платформы контейнеризации за конкретный промежуток времени
18Наличие системы рекомендаций по устранению уязвимостей/некорректной/нежелательной конфигурации
19Наличие готовых CNI плагинов для управления сетевыми интерфейсами контейнеров
20Возможность подключения постоянного хранилища (Persistent Volume)
21Возможность автоматизированного развертывания и масштабирования платформы контейнеризации из интерфейса управления ОП без необходимости ручного создания объектов и редактирования конфигураций (для администратора Платформы)
22Возможность автоматизированного развертывания и масштабирования проекта или экземпляра проекта контейнеризации из интерфейса управления ОП без необходимости ручного создания объектов и редактирования конфигураций (для потребителя Платформы)
23Наличие встроенного в платформу CSI драйвера под используемую платформу контейнеризации
24Возможность управления всеми запущенными сущностями кластеров Kubernetes через единый веб-интерфейс
25Возможность развёртывания приложения через встроенный веб интерфейс (через импорт YAML/JSON файла)
26Наличие встроенного в веб интерфейс YAML редактора для изменения конфигураций манифестов
27Возможность запуска нод Kubernetes одного кластера в разных подсетях
28Наличие модуля управления секретами
29Возможность обновления управляющих сервисов платформы контейнеризации без простоя
30Возможность обновления кластеров Kubernetes без остановки функционирующих приложений
